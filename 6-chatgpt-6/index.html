🔵 1. 소개

- Sysmon이란?

Sysmon은 Microsoft Sysinternals에 의해 개발된 Windows 시스템 모니터링 도구입니다. Sysmon은 시스템 내에서 발생하는 이벤트를 캡처하고 로깅하는 기능을 제공하여 보안 전문가나 시스템 관리자가 시스템의 상태를 모니터링하고 보호하기 위해 사용할 수 있습니다.

Sysmon은 시스템 내에서 프로세스 생성, 파일 작성, 네트워크 연결 등 다양한 이벤트를 모니터링할 수 있습니다. 이러한 이벤트는 Windows 이벤트 로그에 기록되며, 이를 통해 시스템의 이상한 동작이나 보안 위협을 식별할 수 있습니다.

Sysmon은 또한 최신 보안 위협에 대한 정보를 제공하기 위해 기본적으로 설정된 필터 규칙을 가지고 있습니다. 이러한 규칙은 악성 코드 실행, 권한 상승, 암호 해시 작성 등과 같은 일반적인 보안 위협을 식별하는 데 도움을 줍니다.

따라서, Sysmon은 시스템 보안 감사를 강화하고 보안 사고 대응에 도움을 주는 강력한 도구입니다.

- Sysmon의 중요성과 장점

Sysmon은 Microsoft의 Windows 운영 체제에서 실행되는 시스템 모니터링 도구입니다. Sysmon은 시스템 이벤트를 모니터링하고 로깅하여 악성 행위나 보안 위협을 탐지하고 대응할 수 있도록 도와줍니다.

Sysmon의 중요성과 장점은 다음과 같습니다:

  1. 고급 로깅: Sysmon은 시스템 이벤트를 상세하게 기록하므로 보안 전문가들은 시스템의 상태와 악성 행위에 대한 증거를 분석할 수 있습니다. 이는 시스템의 보안 상태를 파악하고 보안 위협을 식별하는 데 도움이 됩니다.

  2. 악성 행위 탐지: Sysmon은 여러 유형의 악성 행위를 탐지할 수 있는 기능을 제공합니다. 예를 들어, 악성 코드 실행, 권한 상승, 계정 탈취, 비정상적인 네트워크 연결 등을 감지할 수 있습니다. 이를 통해 조기 경고를 받고 대응 조치를 취할 수 있습니다.

  3. 이벤트 분석: Sysmon은 이벤트 로그를 생성하므로 보안 전문가들은 이를 분석하여 시스템의 이상 징후를 식별하고 악성 행위를 추적할 수 있습니다. 이는 사고 대응 및 사후 조사를 위한 중요한 자료로 활용될 수 있습니다.

  4. 통합 보안 솔루션과의 통합: Sysmon은 다른 보안 솔루션과 통합하여 전체적인 보안 관리를 강화할 수 있습니다. 예를 들어, SIEM(보안 정보 및 이벤트 관리) 도구와 통합하여 모니터링 및 보안 이벤트 관리를 자동화할 수 있습니다.

요약하면, Sysmon은 Windows 시스템에서 보안 이벤트를 모니터링하고 로깅하여 악성 행위나 보안 위협을 탐지하는 데 도움이 되는 중요한 도구입니다. 이를 통해 시스템 보호 및 보안 감사를 강화할 수 있습니다.

🔵 2. Sysmon 구성 요소

- Event ID

Sysmon은 Microsoft에서 제공하는 Windows 시스템 모니터링 도구입니다. Event ID는 Sysmon에서 생성되는 이벤트를 식별하기 위한 고유한 번호입니다.

Sysmon은 시스템의 활동을 모니터링하고 이를 이벤트로 기록합니다. 이벤트는 시스템 로그에 저장되며, 각 이벤트는 고유한 Event ID를 가지고 있습니다. Event ID는 이벤트의 유형을 식별하여 보안 분석이나 이벤트 로깅에 사용됩니다.

Sysmon에서 생성되는 이벤트들은 다양한 보안 주제에 대한 정보를 제공합니다. 일반적으로 다음과 같은 주제로 분류됩니다:

  1. 프로세스 생성 및 실행 관련 이벤트 (Event ID 1, 7)

    • 새로운 프로세스 생성 및 실행에 대한 정보를 제공합니다. 악성코드나 이상한 동작을 하는 프로세스를 식별하는 데 도움이 됩니다.

  2. 네트워크 통신 관련 이벤트 (Event ID 3)

    • 시스템에서 발생한 네트워크 통신에 대한 정보를 제공합니다. 악성 활동이 있는지를 감지하거나 네트워크 연결 문제를 분석하는 데 도움이 됩니다.

  3. 드라이버 로드 관련 이벤트 (Event ID 6)

    • 시스템에 로드된 드라이버에 대한 정보를 제공합니다. 악성 드라이버 로드나 시스템 안정성 문제를 확인하는 데 도움이 됩니다.

  4. 레지스트리 변경 관련 이벤트 (Event ID 13)

    • 시스템 레지스트리의 변경에 대한 정보를 제공합니다. 악성 스크립트 실행이나 시스템 설정 변경을 모니터링하는 데 도움이 됩니다.

  5. 웹사이트 방문 관련 이벤트 (Event ID 11)

    • 시스템에서 방문한 웹사이트에 대한 정보를 제공합니다. 악성 웹사이트 접근이나 웹사이트 필터링을 분석하는 데 도움이 됩니다.

이외에도 다양한 이벤트들이 있으며, 이러한 이벤트들은 시스템의 보안 감시 및 위협 탐지에 활용될 수 있습니다.

- 필터링 옵션

Sysmon은 Microsoft가 개발한 Windows 운영 체제에서 실행되는 시스템 모니터링 도구입니다. Sysmon은 시스템에 대한 상세한 정보를 수집하고 이를 보안 분석 및 이벤트 로깅을 위해 사용됩니다.

Sysmon에서 필터링 옵션은 수집된 이벤트 데이터를 특정 기준에 따라 필터링하는 기능을 제공합니다. 필터링 옵션을 사용하면 특정 이벤트 유형, 프로세스, 파일 또는 레지스트리 키에 대한 정보만 수집하도록 Sysmon을 구성할 수 있습니다.

예를 들어, 특정 프로세스의 동작 또는 특정 파일의 변경을 추적하고 싶을 때 필터링 옵션을 사용하여 해당 이벤트만 수집할 수 있습니다. 이를 통해 시스템 로그 데이터의 양을 줄이고 중요한 보안 이벤트에 집중할 수 있습니다.

Sysmon 필터링 옵션은 XML 형식의 구성 파일을 사용하여 구성됩니다. 이 구성 파일을 수정하여 원하는 이벤트만 수집하도록 Sysmon을 조정할 수 있습니다. 필터 옵션에는 프로세스 이름, 파일 이름, 레지스트리 경로 등과 같은 다양한 속성을 기반으로 이벤트를 필터링하는 기능이 포함될 수 있습니다.

- 로깅 설정

Sysmon은 Windows 운영 체제에서 실행되는 강력한 보안 도구입니다. 로깅 설정은 Sysmon을 통해 수집된 이벤트 데이터를 어떻게 기록하고 저장할지 결정하는 방법을 말합니다. Sysmon 로깅 설정은 XML 형식으로 제공되며, 이 설정 파일을 사용하여 Sysmon이 수집할 이벤트 유형과 필터링 옵션, 로깅 위치 등을 구성할 수 있습니다.

Sysmon 로깅 설정은 다음과 같은 주요 요소를 포함할 수 있습니다:

  1. 이벤트 유형 (Event Types): 로깅할 이벤트 유형을 지정할 수 있습니다. 예를 들어, 프로세스 생성, 네트워크 연결, 파일 생성 등의 이벤트를 선택할 수 있습니다.

  2. 필터링 (Filtering): 로깅할 이벤트를 세부적으로 필터링하는 옵션을 제공합니다. 예를 들어, 특정 프로세스 이름, 경로, 해시 값에 대한 이벤트만 로깅하도록 필터링할 수 있습니다.

  3. 로깅 위치 (Logging Location): 수집된 이벤트 데이터를 저장할 위치를 설정할 수 있습니다. 이 설정에는 로그 파일의 경로와 이름, 이벤트 데이터를 전송할 원격 서버의 주소 등이 포함될 수 있습니다.

  4. 트리거 액션 (Trigger Actions): 특정 조건이 충족될 때 추가적인 액션을 수행하도록 설정할 수 있습니다. 예를 들어, 특정 이벤트가 발생하면 알림을 보내거나 프로세스를 종료하는 등의 작업을 수행할 수 있습니다.

정확한 로깅 설정은 조직의 보안 요구 사항과 운영 환경에 따라 다를 수 있습니다. 따라서 Sysmon 로깅 설정을 구성할 때는 조직의 보안 정책과 규정을 준수하고 필요한 이벤트를 기록할 수 있도록 설정하는 것이 중요합니다.

🔵 3. Sysmon을 사용한 보안 감사 및 탐지

- 악성 행위 탐지

Sysmon은 Microsoft가 개발한 Windows 시스템 모니터링 도구입니다. Sysmon은 시스템 이벤트를 캡처하고 로깅하여 보안 분석과 악성 행위 탐지에 도움을 줍니다.

악성 행위 탐지는 시스템에 대한 이상 징후를 식별하고 악성 행위를 감지하는 것을 의미합니다. 이를 위해 Sysmon은 시스템 이벤트를 기록하고, 이러한 이벤트를 분석하여 악성 행위를 탐지하는 규칙을 적용합니다. 예를 들어, Sysmon은 프로세스 실행, 네트워크 연결, 파일 생성 및 수정 등의 이벤트를 모니터링할 수 있으며, 이러한 이벤트에 대한 패턴을 설정하여 악성 행위를 탐지할 수 있습니다.

악성 행위 탐지는 시스템의 보안을 강화하고, 공격자의 행위를 사전에 탐지하여 대응하는 데 도움이 됩니다. Sysmon은 이러한 목적으로 사용되며, 다양한 보안 솔루션과 연계하여 효과적인 보안 감시 및 대응을 제공합니다.

- 이상 징후 탐지

Sysmon은 Windows 운영 체제의 시스템 활동을 모니터링하고 기록하는 유틸리티입니다. 이상 징후 탐지는 시스템의 이상한 동작이나 행동을 감지하는 프로세스입니다. 이상 징후 탐지는 보안 공격, 악성 코드, 내부 위협과 같은 사건을 식별하고 이에 대한 경고 또는 대응 조치를 취할 수 있도록 도와줍니다. Sysmon을 사용하여 이상 징후 탐지를 수행하면 시스템에서 발생하는 이벤트를 실시간으로 모니터링하고 분석할 수 있으며, 이상 징후를 탐지하는 데 도움이 될 수 있는 패턴이나 행동을 식별할 수 있습니다. 이를 통해 조기 경고 시스템을 구축하고 보안 위협에 대한 대응을 빠르게 할 수 있습니다.

- 관리자 활동 감사

Sysmon은 Windows 시스템에서 발생하는 이벤트를 기록하는 유틸리티입니다. 이러한 이벤트들은 보안 감사 및 위협 탐지, 시스템 모니터링, 인시던트 대응 등 다양한 보안 관련 작업에 사용될 수 있습니다.

관리자 활동 감사는 시스템 관리자가 시스템에 대한 접근 및 활동을 모니터링하고 기록하는 것을 의미합니다. 이를 통해 시스템 관리자가 시스템에 대한 권한을 남용하지 않고, 불법적인 활동을 감지하거나 인지할 수 있습니다. Sysmon은 관리자 활동 감사를 위해 다양한 이벤트를 기록할 수 있으며, 예를 들어 다음과 같은 관련 정보를 기록할 수 있습니다:

  1. 계정 로그인 및 로그아웃 이벤트
  2. 관리자 권한 상승 이벤트
  3. 시스템 설정 변경 이벤트
  4. 파일 시스템 변경 이벤트
  5. 프로세스 생성 및 종료 이벤트
  6. 네트워크 연결 및 통신 이벤트

이러한 이벤트들은 보안 관리자가 시스템 사용자의 활동을 추적하고, 불법적인 활동이나 악성 코드 실행을 감지할 수 있도록 도와줍니다. 또한, 이벤트 로그를 분석하여 시스템 보안 취약성을 식별하고, 인시던트 대응을 더욱 효과적으로 수행할 수 있습니다.

- 파일 및 레지스트리 변경 감사

Sysmon은 Windows 시스템에서 실행되는 고급 시스템 모니터링 도구입니다. 이 도구는 시스템의 파일 및 레지스트리 변경을 감사하는 기능을 제공합니다.

파일 및 레지스트리 변경 감사는 시스템에서 발생한 변경 사항을 기록하고, 이를 통해 시스템에 대한 보안 이벤트를 모니터링하고 분석하는 것을 의미합니다. 이를 통해 악성 소프트웨어나 해킹 시도 등에 대한 조기 탐지와 대응이 가능해집니다.

Sysmon을 사용하면 파일 시스템 이벤트(파일 생성, 수정, 삭제 등)와 레지스트리 이벤트(레지스트리 키 생성, 수정, 삭제 등)를 실시간으로 기록할 수 있습니다. 이러한 이벤트 로그는 중앙 집중식 로그 관리 시스템에 전송되어 분석 및 보안 이벤트 탐지에 활용될 수 있습니다.

파일 및 레지스트리 변경 감사는 시스템 보안 강화와 위협 탐지에 매우 중요한 역할을 합니다. 이를 통해 시스템 관리자는 시스템에 대한 변경 사항을 추적하고 악의적인 활동을 감지할 수 있으며, 이를 통해 보안 사고의 발생을 예방하고 대응할 수 있습니다.

🔵 4. Sysmon 설정 및 최적화

- 필요한 이벤트 ID 설정

Sysmon은 Windows 시스템에서 실행되는 고급 시스템 모니터링 도구입니다. 이벤트 ID는 Sysmon이 생성하는 이벤트의 식별자입니다. 이벤트 ID 설정은 시스템 모니터링 및 보안 감사를 위해 필요한 이벤트를 지정하는 과정입니다.

보안 주제에서 필요한 이벤트 ID 설정은 주로 다음과 같은 것들을 포함합니다:

  1. 프로세스 생성 및 종료 (Event ID 1, 5): 악성코드 실행 시 프로세스 생성 및 종료를 모니터링하여 악성 행위를 탐지할 수 있습니다.
  2. 파일 생성 및 삭제 (Event ID 11, 12): 악성 파일 생성 및 삭제를 모니터링하여 악성 행위를 탐지할 수 있습니다.
  3. 네트워크 연결 (Event ID 3): 악성 네트워크 연결을 모니터링하여 C&C 서버와의 통신 등을 탐지할 수 있습니다.
  4. 레지스트리 변경 (Event ID 13, 14): 악성 레지스트리 변경을 모니터링하여 시스템 변조 또는 악성 행위를 탐지할 수 있습니다.
  5. 드라이버 로드 (Event ID 6): 악성 드라이버 로드를 모니터링하여 시스템 변조 또는 악성 행위를 탐지할 수 있습니다.

이외에도 보안 주제에 따라 필요한 다양한 이벤트 ID 설정이 가능합니다. 설정된 이벤트 ID를 통해 Sysmon은 해당 이벤트가 발생할 때 알림을 제공하거나 로그를 생성하여 보안 분석에 활용할 수 있습니다.

- 필터링 옵션 활용

Sysmon은 Windows 시스템에서 발생하는 이벤트를 모니터링하고 기록하는 유틸리티입니다. Sysmon을 사용하면 시스템 내부에서 발생하는 보안 이벤트를 실시간으로 확인하고 분석할 수 있습니다.

Sysmon의 필터링 옵션은 사용자가 특정 이벤트를 선택하거나 제외할 수 있도록 도와줍니다. 필터링 옵션을 사용하면 중요하지 않은 이벤트를 무시하고, 보안 이벤트에 집중할 수 있습니다. 필터링 옵션을 사용하면 시스템 로그를 적절하게 관리하고, 성능 문제를 방지할 수 있습니다.

Sysmon의 필터링 옵션에는 다양한 기준이 있을 수 있습니다. 예를 들어, 이벤트 ID, 프로세스 이름, 파일 이름, 명령어 라인 인수 등을 기준으로 이벤트를 필터링할 수 있습니다. 필터링 옵션을 사용하여 원하는 이벤트만을 추출하고, 분석하여 보안 위협을 탐지하고 대응할 수 있습니다.

필터링 옵션은 Sysmon의 구성 파일인 XML 파일을 수정하여 설정할 수 있습니다. XML 파일에서는 필터링 옵션을 설정할 수 있는 다양한 속성과 값을 제공합니다. 사용자는 필터링 옵션을 자신의 필요에 맞게 수정하여 보안 이벤트를 효율적으로 관리할 수 있습니다.

- 저장 및 보관 설정

Sysmon은 Windows 시스템에서 실행되는 감사 도구로, 시스템 활동을 모니터링하고 이벤트를 기록하여 보안 분석 및 사건 조사를 지원합니다. 저장 및 보존 설정은 Sysmon에서 생성된 이벤트 데이터의 저장 및 보관 방법을 제어하는 기능입니다.

Sysmon의 저장 및 보존 설정은 다음과 같은 요소를 포함할 수 있습니다:

  1. 저장 경로: Sysmon 이벤트 데이터를 저장할 디렉토리를 지정합니다. 예를 들어, C:\Sysmon\Logs와 같은 경로를 설정할 수 있습니다.

  2. 파일 크기 제한: Sysmon 이벤트 데이터 파일의 크기를 제한합니다. 이 크기에 도달하면 새 파일이 생성됩니다. 일반적으로 파일 크기를 제한하여 데이터의 관리 용이성을 높일 수 있습니다.

  3. 파일 보관 기간: Sysmon 이벤트 데이터를 보존하는 기간을 설정합니다. 지정된 기간 동안 보관된 데이터는 삭제되지 않고 유지됩니다. 일반적으로 보안 정책 및 규정에 따라 데이터 보존 기간을 설정합니다.

  4. 이벤트 필터링: 특정 이벤트 유형만 저장하도록 Sysmon을 구성할 수 있습니다. 예를 들어, 특정 프로세스 또는 파일 활동과 관련된 이벤트만 저장하도록 필터링할 수 있습니다.

  5. 이벤트 형식: Sysmon 이벤트 데이터의 형식을 지정할 수 있습니다. XML 또는 JSON과 같은 형식을 선택할 수 있습니다.

이러한 저장 및 보존 설정을 통해 Sysmon은 중요한 보안 정보를 저장하고 필요한 경우에 이를 검토하고 분석할 수 있도록 합니다.

  2. 로그 시각화 도구 사용: 시각화 도구를 사용하여 Sysmon 로그를 시각화합니다. 예를 들어, Elastic Stack, Splunk, Kibana 등의 도구를 사용하여 로그 데이터를 시각적으로 표현할 수 있습니다. 이를 통해 이상 징후를 식별하고, 보안 위협을 탐지할 수 있습니다.

  4. 이벤트 상관 관계 분석: Sysmon 로그는 시스템 내부에서 발생하는 다양한 이벤트를 기록하므로, 이벤트 간의 상관 관계를 분석하여 보안 위협을 조기에 탐지할 수 있습니다. 예를 들어, 악성 코드 실행과 관련된 이벤트들을 연결하여 이상 징후를 탐지하거나, 특정 이벤트 패턴을 찾아내는 등의 분석을 수행할 수 있습니다.

이러한 방법들을 통해 Sysmon 로그를 시각화하고 분석함으로써, 시스템 내부에서 발생하는 보안 위협을 탐지하고 대응할 수 있습니다.

- Sysmon 설정 시 고려해야 할 사항

Sysmon은 Microsoft에서 제공하는 Windows 시스템 모니터링 도구로써, 시스템에 대한 로깅 및 보안 이벤트 수집을 위해 사용됩니다. Sysmon 설정 시 고려해야 할 주요 사항은 다음과 같습니다.

  1. 필요한 이벤트 필터링: Sysmon은 다양한 시스템 이벤트를 수집할 수 있으며, 이벤트 필터를 설정하여 필요한 이벤트만 수집하도록 구성해야 합니다. 필요한 이벤트는 보안 요구사항과 조직의 운영 환경에 따라 다를 수 있습니다.

  2. 이벤트 저장 및 분석: Sysmon은 이벤트를 로그 파일로 저장하거나 보안 정보 및 이벤트 관리 시스템(SIEM)과 같은 중앙 집중식 로깅 시스템으로 전송할 수 있습니다. 이러한 이벤트 저장 및 분석 시스템을 구성하여 수집한 이벤트를 검토하고 분석할 수 있어야 합니다.

  3. 보안 이벤트 설정: Sysmon은 시스템에 대한 다양한 보안 이벤트를 수집할 수 있습니다. 예를 들어, 프로세스 생성, 네트워크 연결, 파일 생성 및 수정 등의 이벤트를 수집할 수 있습니다. 이러한 보안 이벤트 설정은 조직의 보안 정책 및 요구사항에 따라 구성되어야 합니다.

  4. 이벤트 모니터링 및 대응: Sysmon으로 수집한 이벤트를 실시간으로 모니터링하고 적절한 대응 조치를 취할 수 있어야 합니다. 이를 위해 SIEM과 같은 보안 이벤트 관리 시스템을 사용하거나, 이벤트 모니터링 도구를 활용할 수 있습니다.

  5. 보안 정책 및 규정 준수: Sysmon 설정은 조직의 보안 정책 및 규정을 준수해야 합니다. 예를 들어, 개인정보 보호법이나 산업 규제와 같은 규정에 따라 필요한 이벤트 수집 및 보안 요구사항을 충족시켜야 합니다.

  6. 이벤트 기록 및 보안 사고 대응: Sysmon 설정은 보안 사고 대응을 위한 이벤트 기록을 가능하게 해야 합니다. 이를 통해 시스템에 대한 이상 징후를 탐지하고 보안 사고 대응에 필요한 정보를 제공할 수 있습니다.

이러한 사항들을 고려하여 Sysmon을 설정하면 시스템 보안 강화 및 보안 사고 대응 능력을 향상시킬 수 있습니다.

- 중요한 이벤트 ID 및 행위 탐지 패턴

Sysmon은 Windows 시스템에서 실행되는 프로세스 및 네트워크 활동을 모니터링하는 도구입니다. Sysmon을 사용하여 중요한 보안 이벤트 및 행위를 탐지할 수 있습니다.

중요한 이벤트 ID 중 일부는 다음과 같습니다:

  1. Event ID 1: 프로세스 생성 - 새로운 프로세스가 생성되었을 때 기록됩니다. 악성코드 실행 및 권한 상승 공격에 사용될 수 있습니다.

  2. Event ID 3: 네트워크 연결 - 시스템이 외부 서버 또는 장치와의 네트워크 연결을 시도할 때 기록됩니다. C&C 서버와의 통신 또는 데이터 노출을 시도하는 악성코드를 탐지하는 데 사용될 수 있습니다.

  3. Event ID 7: 이미지 로딩 - 실행 가능한 파일이 메모리에 로드될 때 기록됩니다. 악성 DLL 삽입 또는 프로세스 감염과 같은 공격을 감지할 수 있습니다.

  4. Event ID 11: 드라이버 로드 - 시스템에 설치된 드라이버가 로드될 때 기록됩니다. 악성 드라이버를 감지하는 데 사용될 수 있습니다.

  5. Event ID 13: 서비스 생성 - 새로운 서비스가 생성되었을 때 기록됩니다. 악성 서비스 설치 또는 권한 상승을 시도하는 공격을 탐지하는 데 사용될 수 있습니다.

중요한 행위 탐지 패턴 중 일부는 다음과 같습니다:

  1. 새로운 프로세스 생성 및 실행
  2. 시스템 파일 변경 또는 삭제
  3. 레지스트리 변경
  4. 비정상적인 네트워크 연결 또는 데이터 전송
